# 用户故事模板

> 使用说明(给执行的 AI):用于把需求拆成可执行的用户故事。每个故事必须通过 INVEST 检查并带验收
> 标准。粒度过大(无法在一个迭代内完成)→ 标记为史诗(Epic)并拆分。默认中文,写得直白少术语。
> 用完即删本说明。

- 拆分思路:优先打通一条端到端的"走通骨架"(端到端最小闭环),再逐步加深。

## US-[编号]:[故事简述]

**故事**
> 作为 **[用户角色]**,我希望 **[目标/操作]**,以便 **[价值/收益]**。

**优先级(MoSCoW):** 必须做 / 应该做 / 可以做 / 本期不做
**估算(故事点):** _(斐波那契数列:1,2,3,5,8,13)_

**验收标准**(覆盖正常 + 边界 + 异常三类;每条写出系统**应有的行为**,而非"妥善处理")
- **正常路径:** 前置[条件], 操作[动作], 预期[可观察结果]。
- **边界:** 前置[临界条件, 如取值为 0 / 上限 / 上限+1 / 空列表], 操作 …, 预期 …。
- **异常:** 前置[失败条件, 如无权限 / 超时 / 重复提交], 操作 …, 预期[拦截 / 降级 / 重试 + 提示什么]。

**边界与异常自检**(逐项过一遍, 只留与本故事相关的, 各写出预期行为;方法见 `references/edge-cases-and-exceptions.md`)
- [ ] 空 / 缺失输入  [ ] 零 / 一个 / 多个(空列表、仅一条、海量)  [ ] 数值与长度边界(最小、最大、上限+1、负数、超长)
- [ ] 非法格式 / 特殊字符  [ ] 重复 / 冲突提交  [ ] 并发 / 时序(竞态、重试)  [ ] 权限 / 状态不符
- [ ] 外部依赖失败(超时、第三方报错、网络中断、部分成功)  [ ] 时间 / 时区 / 边界日期

**技术任务与依赖**
- [ ] 任务 1
- [ ] 任务 2
- 依赖:[前置故事/接口/数据]

**INVEST 自检**(一条好故事应满足:独立、可协商、有价值、可估算、够小、可测试)
- [ ] 独立 [ ] 可协商 [ ] 有价值 [ ] 可估算 [ ] 一个迭代内可完成 [ ] 可测试

> **完成的定义** 团队统一的质量底线,适用于所有故事:代码完成、测试通过、集成成功、文档齐全、可部署。
> (与单个故事的验收标准区分:验收标准验证某个功能本身,完成的定义是全员通用的交付门槛。)
