# PRD 模板

> 使用说明(给执行的 AI):按本模板产出 PRD,默认中文。小型功能可只保留标注 **[必填]** 的部分作为
> 一页纸 PRD;重大功能用完整模板。所有需求必须可验证;凡是推断出来的假设,显式标注 `（假设）`。
> 写得清楚直白,少用术语;确需保留的术语,首次出现时用一句话解释。用完即删本说明。

## 6. 技术考量
- 方案概述与关键技术决策:
- 依赖系统/接口/数据:
- 性能、安全、合规约束:
- 技术债与权衡(投入与收益):

> 功能含接口/跨系统/调用服务时,这里要落到**契约**而非泛泛而谈,否则工程无法开工。展开方法见
> `references/technical-artifacts.md`:
> - **接口契约**:每个端点的入参/出参表(字段·类型·必填·格式/取值/可空含义)+ 一个真实的请求或响应示例。
> - **数据契约**:每个字段的**输出方/源头**、ID 回传、日期/单位/编码格式、版本演进方式。
> - **错误码分类**:区分**系统级异常**(服务无法完成,返回非 2xx,需兜底)与**业务级警告**(返回成功但
>   标记需人工复核 `needs_review`,结果可用但需人工确认);各列触发条件与下游处理。
> - **把"快/稳/安全"写成数字**:即把非功能需求变成可验证的标准(如 p95 延迟、体积上限、超时、可用性、合规可观测项)。
> - **边界清单**:超大/空/加密/损坏/缺上游数据/重试幂等/部分成功——逐项定"处理 / 拒绝 / 降级"。

### 若本功能由 AI/模型驱动(可选)
产出是概率分布而非确定结果,需补充以下内容(展开见 `references/ai-ml-products.md`):
- **评测口径**:评测集(标注集)、用准确率还是精确率/召回率、阈值;无法测量则首要任务是先埋点。
- **错误代价不对称**:错放(放过坏的)与错拦(拦住好的)谁更贵 → 决定优化目标与分项指标(如误报率/漏报率分开定)。
- **人工兜底**:置信度阈值分流、需人工复核的设计及其在护栏指标中的占比;低置信度返回空值 + 原因,不臆造。
- **数据/规则依赖**:标注集、映射/规则表是**排期前置门槛**(缺它就不能开工),挂负责人与目标时间。
